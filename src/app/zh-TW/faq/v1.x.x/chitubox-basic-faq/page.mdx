## Windows 操作系統問題

<Admonition type="note" title="注意">
	請確保你已經安裝了<NextLink href="/zh-TW/chitubox-basic/v1.9.5/requirements">所需運行環境</NextLink>中提到的所有環境。

### MSVCR110.dll 丟失

“MSVCR110.dll Missing”表示您缺少 Visual Studio 2012 的 Visual C++ Redistributable Packages，請從<SpanA href="https://www.microsoft.com/en-us/download/details.aspx?id=30679">此處</SpanA>下載所需的環境。

### MSVCR120.dll 丟失

“MSVCR120.dll Missing”表示您缺少 Visual Studio 2013 的 Visual C++ Redistributable Packages，請從<SpanA href="https://www.microsoft.com/en-us/download/details.aspx?id=40784">此處</SpanA>下載所需的環境。

### MSVCR140.dll 丟失

“MSVCR140.dll Missing”表示您缺少 Visual Studio 2015，2017，和 2019 的 Visual C++ Redistributable Packages，請從<SpanA href="https://docs.microsoft.com/en-US/cpp/windows/latest-supported-vc-redist?view=msvc-170">此處</SpanA>下載所需的環境。

### 其他系統級錯誤

-   錯誤代碼 -1073741701

-   錯誤代碼 -1073741515

-   錯誤代碼 -1073740791

-   錯誤代碼 -0xc000007b

出現此類問題的原因可能是某些系統檔案損壞或操作系統檔案損壞。

您可以嘗試以下方法進行修復：

方法 1： 在 PC 上打開 cmd（使用 Win + R 並輸入 cmd），輸入以下命令：

```shell
DISM.exe /Online /Cleanup-image /Scanhealth

DISM.exe /Online /Cleanup-image /Restorehealth
```

方法 2： 在 PC 上打開 cmd（使用 Win + R 並輸入 cmd），輸入以下命令：

```shell
sfc /scannow
```

如果這些方法都不能解決問題，您可能需要從微軟官方網站下載並重新安裝您的系統。

-   <SpanA href="https://www.microsoft.com/en-us/software-download/windows11">
    	Windows 11
    </SpanA>

-   <SpanA href="https://www.microsoft.com/en-us/software-download/windows10">
    	Windows 10
    </SpanA>

### Windows 檔案總管反復重啟

如果您的 Windows 檔案總管一直反復重啟，請通過以下方式修復：

按 Ctrl + Shift + Esc 調出您的 Windows 工作管理員，右鍵點擊 Windows 檔案總管，然後結束工作（注意不是重新啟動）。

<ImageContainer src={"/images/docs/zh-CN/faq/chitubox-basic/faq002.png"} />

在工作管理員中，點擊 檔案 > 建立新工作，輸入 `cmd`, 勾選 **以系統管理權限建立此工作**, 然後點擊 **確定**.

輸入：

```shell
regsvr32 /u "CHITUBOX-Installation-Path\CHITUBOX_Thumbnail.dll"
```

注意，這裡要把 `CHITUBOX-Installation-Path` 換成你的 CHITUBOX 安裝路徑，例如：

```shell
regsvr32 /u "C:\Program Files\ChiTubox\CHITUBOX_Thumbnail.dll"
```

按下回車鍵 Enter 執行命令。

<Admonition type="note" title="注意">
	你的 OpenGL 版本不應低於 2.0。
</Admonition>

### 和 Fusion 360 衝突或者 OpenGL 報錯 Failed to create OpenGL context for format QSurfaceFormat

<ImageContainer src={"/images/docs/zh-CN/faq/chitubox-basic/faq001.png"} />

請打開 系統設置 > 關於 > 高級系統設置，刪除環境變數 `QT_OPENGL` 。

### 報錯：This application failed to start because no Qt platform plugin could be initialized

<ImageContainer src={"/images/docs/zh-CN/faq/chitubox-basic/faq006.png"} />

請打開 系統設置 > 關於 > 高級系統設置，刪除 `QT_OPENGL` `QT_PLUGIN_PATH` 這兩個環境變數。

### CHITUBOX 無響應

<ImageContainer src={"/images/docs/zh-CN/faq/chitubox-basic/faq008.png"} />

極有可能是軟體衝突問題，請逐一退出正在運行的軟體，並查找與 CHITUBOX 衝突的軟體。完成此操作後，大多數情況下應該可以解決問題。

以下軟體可能會導致衝突：

-   殺毒軟體，各種管家，衛士，助手，大師

-   Tabtip (在工作欄中稱為 Touch Keyboard and Handwriting Panel)

-   PowerToys

-   Google Drive

## 插件問題

### Longer 錯誤代碼 -1

1. Longer 的控制板對分辨率進行了特殊處理，我們已經確認了正確的分辨率 x：960，y：1708。我們已經仔細檢查並確認了這一點，所以請不要更改它。

2. `錯誤代碼 -1` 更改分辨率後是插件內碼錯誤。 因為您更改了分辨率，所以發生了報錯。

點擊<SpanA href="https://drive.google.com/drive/u/0/folders/1rPRw68b3Z1oqLwc7hSmP5A9Zx0HjatbE?fbclid=IwAR2XyvSs_jCWwqXymOh9ZEvcuyKNuj-ibnmbLAmfOsZMWzjJdke71kxANwQ">此處</SpanA>下載 Longer 插件。

### 切片檔案模型表面粗糙

在模型被切片後，大家可能會覺得圖像的品質很粗糙，但實際上這是模型在切片後透過立體渲染將每一顆體素都表現出來的樣子。就如同大家在 Photoshop 中將圖像放大可以看到每一顆像素的樣子一般。

如圖所示，每一顆立方體的高是在切片設定中設定的層高，長是螢幕上橫向每個像素的長度，寬是螢幕上縱向每個像素的寬度。長、寬、高構成體素的體積。體素是構成列印件的最小單位。

### 為什麼 STL 檔案匯入 CHITUBOX 後比例被縮放了？

STL 檔案並不包含具體的單位資訊，而是使用數字記錄。因此，不同的軟體讀取 STL 檔案時使用的單位不一定相同。例如，當使用 STL 時，Blender 中的 1 公尺相當於 CHITUBOX 中的 1 毫米。您需要先測試您使用的 3D 建模軟體和 CHITUBOX 的比例關係後再匯出檔案。

## 其他問題

出於某種原因，您可能會遇到由版本之間的資料損壞引起的其他問題。您可以嘗試以下方法來解決問題：

### 清除緩存

1. 開啟軟體日誌路徑（設定/幫助 > 打開日誌路徑）。<NextLink href="/zh-TW/chitubox-basic/v1.9.5/application-overview/file#打開日誌路徑">📃 點擊這裡</NextLink> 學習如何打開日誌路徑。

1. 關閉 CHITUBOX Basic (**務必先關閉 CHITUBOX Basic**)。

1. 確保已經關閉 CHITUBOX Basic 後，備份 📁*machine* 資料夾到其他路徑，刪除除了 📄*userLogin.cfg* 以外的所有檔案和資料夾。

    <ImageContainer src={"/images/docs/zh-CN/faq/chitubox-basic/faq003.png"} />

1. 重新打開 CHITUBOX Basic。

1. 重新開啟軟體後，會產生新的 📁*machine* 資料夾，可以將 _.cfg_ 檔案複製粘貼到新的 📁 _machine_ 資料夾中以恢復您的機器。請不要一次性將所有的 _.cfg_ 檔案複製到 📁 _machine_ 資料夾中，因為您遇到的問題有可能是其中一個檔案引起的。

## 隨身碟問題

您可能會發現您的印表機有時無法正確讀取您的隨身碟。這主要是因為您的印表機控制板設計為使用 USB 2.0 且容量等於或小於 16GB。USB 3.0 使用較大的電流，因此有時會對控制板造成電氣干擾。您的隨身碟應滿足以下要求：

-   FAT32 格式
-   USB 2.0
-   ≤ 16GB
-   分配單元大小 = 2048 bytes
