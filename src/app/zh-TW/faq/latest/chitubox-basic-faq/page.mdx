## Windows 操作系統問題

### Windows 檔案總管反復重啟

如果您的 Windows 檔案總管一直反復重啟，請通過以下方式修復：

按 Ctrl + Shift + Esc 調出您的 Windows 工作管理員，右鍵點擊 Windows 檔案總管，然後結束工作（注意不是重新啟動）。

<ImageContainer src={"/images/docs/zh-CN/faq/chitubox-basic/faq002.png"} />

在工作管理員中，點擊 檔案 > 建立新工作，輸入 `cmd`, 勾選 **以系統管理權限建立此工作**, 然後點擊 **確定**.

輸入：

```shell
regsvr32 /u "CHITUBOX-Installation-Path\CHITUBOX_Thumbnail.dll"
```

注意，這裡要把 `CHITUBOX-Installation-Path` 換成你的 CHITUBOX 安裝路徑，例如：

```shell
regsvr32 /u "C:\Program Files\ChiTubox\CHITUBOX_Thumbnail.dll"
```

按下回車鍵 Enter 執行命令。

<Admonition type="note" title="注意">
	你的 OpenGL 版本不應低於 2.0。
</Admonition>

### 和 Fusion 360 衝突或者 OpenGL 報錯 Failed to create OpenGL context for format QSurfaceFormat

<ImageContainer src={"/images/docs/zh-CN/faq/chitubox-basic/faq001.png"} />

請打開 系統設置 > 關於 > 高級系統設置，刪除環境變數 `QT_OPENGL` 。

### 報錯：This application failed to start because no Qt platform plugin could be initialized

<ImageContainer src={"/images/docs/zh-CN/faq/chitubox-basic/faq006.png"} />

請打開 系統設置 > 關於 > 高級系統設置，刪除 `QT_OPENGL` `QT_PLUGIN_PATH` 這兩個環境變數。

### CHITUBOX 無響應

<ImageContainer src={"/images/docs/zh-CN/faq/chitubox-basic/faq008.png"} />

極有可能是軟體衝突問題，請逐一退出正在運行的軟體，並查找與 CHITUBOX 衝突的軟體。完成此操作後，大多數情況下應該可以解決問題。

以下軟體可能會導致衝突：

-   殺毒軟體，各種管家，衛士，助手，大師

-   Tabtip (在工作欄中稱為 Touch Keyboard and Handwriting Panel)

-   PowerToys

-   Google Drive

## 插件問題

### Longer 錯誤代碼 -1

1. Longer 的控制板對分辨率進行了特殊處理，我們已經確認了正確的分辨率 x：960，y：1708。我們已經仔細檢查並確認了這一點，所以請不要更改它。

2. `錯誤代碼 -1` 更改分辨率後是插件內碼錯誤。 因為您更改了分辨率，所以發生了報錯。

### 切片檔案模型表面粗糙

在模型被切片後，大家可能會覺得圖像的品質很粗糙，但實際上這是模型在切片後透過立體渲染將每一顆體素都表現出來的樣子。就如同大家在 Photoshop 中將圖像放大可以看到每一顆像素的樣子一般。

如圖所示，每一顆立方體的高是在切片設定中設定的層高，長是螢幕上橫向每個像素的長度，寬是螢幕上縱向每個像素的寬度。長、寬、高構成體素的體積。體素是構成列印件的最小單位。

### 為什麼 STL 檔案匯入 CHITUBOX 後比例被縮放了？

STL 檔案並不包含具體的單位資訊，而是使用數字記錄。因此，不同的軟體讀取 STL 檔案時使用的單位不一定相同。例如，當使用 STL 時，Blender 中的 1 公尺相當於 CHITUBOX 中的 1 毫米。您需要先測試您使用的 3D 建模軟體和 CHITUBOX 的比例關係後再匯出檔案。

## 其他問題

出於某種原因，您可能會遇到由版本之間的資料損壞引起的其他問題。您可以嘗試以下方法來解決問題：

### 清除緩存

1. 請打開目錄：

    - Windows:

    ```shell
    C:\Users\{user-name}\AppData\Local
    ```

    - Mac:

    ```shell
    Macintosh HD/Users/{user-name}/Library/Application Support
    ```

1. 關閉 CHITUBOX Basic (**務必先關閉 CHITUBOX Basic**)。

1. 確保已經關閉 CHITUBOX Basic 後，找到該目錄下的 📂chitubox2_0 和 📂chituboxResource 文件夾，將這兩個文件夾刪除。

    <ImageContainer
    	src={"/images/docs/zh-CN/faq/chitubox-basic/2.x.x/windows-path.png"}
    />

1. 重新打開 CHITUBOX Basic。

<Admonition type="warning" title="警告">
	清除緩存將會刪除您的機器設定，請確保在清除緩存前進行了備份。
</Admonition>

## 隨身碟問題

您可能會發現您的列印機有時無法正確讀取您的隨身碟。這主要是因為您的列印機控制板設計為使用 USB 2.0 且容量等於或小於 16GB。USB 3.0 使用較大的電流，因此有時會對控制板造成電氣干擾。您的隨身碟應滿足以下要求：

-   FAT32 格式
-   USB 2.0
-   ≤ 16GB
-   分配單元大小 = 2048 bytes

## 網路傳送問題

如果您在使用網路傳送功能時遇到問題，例如找不到列印機、檔案傳送失敗、傳輸進度條緩慢等，可以檢查以下原因：

-   網路連線：

    確保您的電腦和列印機連接到同一區域網路 (LAN)。

-   防火牆設定：

    檢查您的防火牆是否攔截了 ChituManager， 如果被攔截，將無法識別到您的列印機。

-   檔案名稱字元：

    避免在檔案名稱中使用非英文、非數字的字元。 否則，有可能導致檔案傳輸失敗。
