# 設定列印參數

要設定列印參數，你需要先添加一台列印機。如果你還未添加列印機，請點擊 <NextLink href="/zh-CN/docs/chitubox-pro/latest/application-overview/start#添加機器">此處</NextLink> 查看如何添加一台列印机。

##### 2.2 燈滅延遲

###### 2.2.1 什麽是燈滅延遲

**燈滅延遲** 指的是在 3D 列印過程中，列印一層時在列印平台上 **抬升**，**等待** 和 **回程** 所花費的總時間。

在開始列印下一層之前，確保樹脂有充足的穩定固化時間是非常重要的。大多數情況下，燈滅延遲的時間設定得越久，得到的列印結果就越好。但是如果你設定得太久，就會耗費很多時間成本。

**抬升** 和 **回程** 由抬升距離決定，而 **等待** 則不同，一般情況下我們推薦 2 到 3 秒。

###### CHITUBOX 中的燈滅延遲計算

為了計算燈滅延遲，你需要知道抬升距離，抬升速度以及回程速度。算出抬升和回程的總時間，再加上額外 2 到 3 秒的等待時間，你就得到了燈滅延遲時間。

燈滅延遲 = 抬升時間 + 回程時間 + 等待時間

舉個例子：

<ImageContainer
	src={
		"/images/docs/zh-CN/chitubox-pro/1.x.x/004-setting-up-configure-printer-parameters003.2.png"
	}
/>

這裏 抬升距離 是 7mm，抬升速度 是 70mm/min (1.167mm/sec)， 因此抬升時間 $T_L$ 為：

$$T_L = \frac{7}{1.167} = 5.998\ \text{秒}$$

類似地，回程速度 是 210mm/min (3.5mm/sec), 回程時間 $T_R$ 是

$$T_R = \frac{7}{3.5} = 2\ \text{秒}$$

如果我們希望最小等待時間比 2s 長，那麽燈滅延遲 $T_{LOD}$ 則應該為：

$$T_{LOD} \geq 5.998 + 2 + 2 \approx 10\ \text{秒}$$

如果你在燈滅延遲中輸入一個小於 $ { T_L + T_R} $ 的數，則燈滅延遲不工作。

圖像灰度等級，類似光強 PWM，但圖像灰度不改變 UV 光強，它改變的是 LCD 屏幕的透光率。

#### 4. 抗鋸齒

對圖像邊緣的三角形鋸齒進行柔化處理以實現物體更光滑的表面效果。相鄰三角形鋸齒間通過增加或刪除一定灰度的像素單元可以調控邊緣曝光的連續性和透光強度，進而實現邊緣區域更加光滑的效果。

-   _灰度範圍_：`0` ~ `255`， `0` 代表黑色， `255` 代表白色。**適用於 <i>.ctb</i> 格式**，它決定了可用於抗鋸齒的灰度範圍。

    圖像邊緣三角形鋸齒進行柔化處理時的灰度範圍的設定，與 CHITUBOX Basic 中灰度級別的轉化公式如下：

    默認灰度範圍的第一個數值 = $$\frac{256}{8}\times n - 1 \text{（n = 1, 2 , 3…，8）}$$

    第二個數值均為 255。當級別為 n = 0 時對應第一個數值的灰度值為 0。

    舉個例子：
    如果您在 Basic 版本中設置的灰度級別為 2（當級別為 n = 2），想要在 Pro 版本中達到同樣的效果，那麼相對應的第一個數值的灰度值為：

    $$\frac{256}{8} \times 2 - 1 \ = 63$$

    | Basic(灰度級別) | Pro(灰度範圍) | 顔色參考                          |

	添加禁用底部公差補償時，普通層公差補償對全模型生效。
</Admonition>

#### 8. 列印時間補償

部分高解析度列印機在載入數據時可能會產生額外的延遲，從而導致預估列印時間不準確。可以通過設定列印時間補償來改善這個問題。

-   **手動輸入**

    層列印時間補償: 一個在 `-99.99` 和 `99.99` 之間的 2 位數，預設為 `0` 秒。

    當啟用時，

    單層的預估列印時間 = 曝光時間 + 等待時間 + 運動時間 + 層列印時間補償

-   **自動生成**

    <ImageContainer
    	src={
    		"/images/docs/zh-CN/chitubox-pro/1.x.x/004-setting-up-configure-printer-parameters004-1.png"
    	}
    />

    $$\text{層列印時間補償} = \frac{\text{實際列印時間} - \text{軟件預測時間}}{\text{列印總層數}}$$

<Admonition type="note" title="注意">
	設定完參數請點擊<b>應用</b>，否則參數不生效。
</Admonition>

#### 9. 蒙版

蒙版是爲了解決一個常見的光源問題而設計的：光源在屏幕上不同位置的光強不均勻，這通常是由於使用了不合適的光源所致，蒙版的目的是通過調節屏幕不同區域的透光率從而提供一層修正層，使得原本不均勻的光源變得均勻。

_蒙版文件_：您可以通過點擊 `...` 圖標來選擇現有的文件，或者點擊齒輪圖標以使用 CHITUBOX 內置的 <NextLink href="#91-蒙版生成器">蒙版生成器</NextLink> 來生成一張蒙版圖片。<br />

##### 9.2 灰度單位

**灰度單位** 的原理是在 X 軸方向上對指定數量像素的灰度取平均值並應用到對應的像素上。

舉個例子：

假設列印平台的尺寸是 80mm × 120mm，屏幕的分辨率為 800px × 1200px。

下圖為蒙版圖，分辨率也是 800px × 1200px。其中所有的小方塊的分辨率均為 50px × 50px。

<ImageContainer
	src={
		"/images/docs/zh-CN/chitubox-pro/1.x.x/004-setting-up-configure-printer-parameters004.3.png"
	}
	style={{ width: 400 }}
/>

如果將 **灰度單位** 設爲 51px, 則處理后的圖像將如下圖所示：

<ImageContainer
	src={
		"/images/docs/zh-CN/chitubox-pro/1.x.x/004-setting-up-configure-printer-parameters004.4.png"
	}
	style={{ width: 400 }}
/>

灰度從左到右逐漸變灰，這是因為原圖上的每 51 個像素灰度被平均了。

考慮第一行的情况。第一組 51 像素的灰度是 50 個黑像素和 1 個白像素的平均值，因此結果會比黑色略淺一點點，雖然這個改变几乎肉眼不可見。第二組 51 像素的灰度是 49 個白像素和 2 個黑像素的平均值，因此結果會比白色略暗一點點。第三組 51 像素的灰度是 48 個黑像素和 3 個白像素的平均值，依此類推，直到剩下的餘數 35 個白像素。

$$\frac{800}{51} = 15$$，餘數 35

這個部分則由它们自己取平均值，因此灰度沒有变化。
